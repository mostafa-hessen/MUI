ما هي Material-UI (MUI)؟

Material-UI هي مكتبة مكونات واجهة مستخدم (UI Components) شائعة ومفتوحة المصدر مبنية على React وتتبع إرشادات تصميم Material Design من Google.

ما الذي تعتمد عليه MUI؟

  • React

    المكتبة الأساسية لبناء واجهات المستخدم

  • Emotion/Styled Components

    لمكونات CSS-in-JS

  • Material Design System

    نظام التصميم من Google

MUI

أمثلة على مكونات MUI: أزرار، حقول إدخال، بطاقات، أشرطة تنقل

المنافسون والفروقات

المكتبة نقاط القوة نقاط الضعف أفضل استخدام
Material-UI تصميم جميل، وثائق ممتازة، مجتمع كبير، قابل للتخصيص حجم الحزمة كبير، منحنى تعلم متوسط تطبيقات المؤسسات، تطبيقات Material Design
Ant Design مجموعة مكونات غنية، تصميم احترافي، دعم دولي ممتاز تخصيص أقل، ثقيلة الحجم لوحات التحكم، التطبيقات التجارية
Chakra UI سهولة الاستخدام، قابلية الوصول، قابلية للتكوين مكونات أقل، مجتمع أصغر تطبيقات الويب الحديثة، البدء السريع

متى تختار MUI؟

  • عندما تحتاج تطبيقًا بجودة احترافية
  • عندما تريد تصميم Material Design
  • عندما تحتاج إلى الكثير من المكونات الجاهزة

متى تتجنب MUI؟

  • عندما تحتاج تطبيقًا خفيفًا جدًا (حجم الحزمة)
  • عندما لا تريد تصميم Material Design
  • عندما تعمل في مشروع صغير جدًا وسريع

تثبيت MUI مع Emotion

لماذا Emotion؟

Emotion هي مكتبة CSS-in-JS تتيح كتابة أنماط CSS داخل JavaScript. MUI تستخدمها بشكل افتراضي بدءًا من الإصدار 5.

مميزات Emotion:

  • أداء أفضل من styled-components
  • حجم حزمة أصغر
  • دعم أفضل لخواص CSS الحديثة

كيف تعمل؟

  • توليد أنماط ديناميكية
  • تحسين أداء التطبيق
  • تجنب تعارض الأسماء

خطوات التثبيت

1. إنشاء مشروع React جديد (إن لم يكن موجودًا)

npx create-react-app my-app
cd my-app

2. تثبيت حزم MUI الأساسية

npm install @mui/material @emotion/react @emotion/styled

3. تثبيت أيقونات MUI (اختياري)

npm install @mui/icons-material

4. تهيئة Theme الأساسي (في App.js)

import { ThemeProvider, createTheme } from '@mui/material/styles';
import Button from '@mui/material/Button';

// إنشاء تخصيص للتصميم
const theme = createTheme({
  palette: {
    primary: {
      main: '#1976d2',
    },
    secondary: {
      main: '#9c27b0',
    },
  },
});

function App() {
  return (
    <ThemeProvider theme={theme}>
      <Button variant="contained" color="primary">
        زر MUI
      </Button>
    </ThemeProvider>
  );
}

Emotion مقابل Styled Components

Emotion

نقاط القوة:

  • حجم حزمة أصغر (حوالي 7KB مقابل 12KB)
  • أداء أفضل في بعض السيناريوهات
  • تركيب أسهل مع MUI
import { css } from '@emotion/react';
import { styled } from '@mui/material/styles';

// إنشاء مكون مخصص
const CustomButton = styled(Button)`
  background: linear-gradient(45deg, #FE6B8B 30%, #FF8E53 90%);
  border: 0;
  color: white;
  padding: 10px 20px;
  border-radius: 8px;
`;

// استخدام أنماط مباشرة
const styles = css`
  background-color: hotpink;
  &:hover {
    color: white;
  }
`;

Styled Components

نقاط القوة:

  • مجتمع أكبر وشعبية أوسع
  • وثائق ممتازة وأمثلة كثيرة
  • أكثر نضجًا واستقرارًا
import styled from 'styled-components';

const CustomButton = styled.button`
  background: linear-gradient(45deg, #FE6B8B 30%, #FF8E53 90%);
  border: 0;
  color: white;
  padding: 10px 20px;
  border-radius: 8px;
  
  &:hover {
    opacity: 0.9;
    transform: translateY(-2px);
  }
`;

// استخدام مع المكونات
<CustomButton>زر مخصص</CustomButton>

ما الذي تختاره لمشروعك؟

اختر Emotion عندما:

  • تستخدم MUI (فهي الخيار الافتراضي)
  • تحتاج إلى حجم حزمة أصغر
  • تريد أداءً أفضل في التطبيقات الكبيرة

اختر Styled Components عندما:

  • تعمل في مشروع غير مرتبط بـ MUI
  • تحتاج إلى مجتمع دعم كبير
  • تستخدم خواص متقدمة في CSS-in-JS

Typography - النصوص

مكون Typography في MUI يستخدم لعرض النصوص بشكل منسق وجميل. يتضمن مجموعة متنوعة من الـvariants لتتناسب مع مختلف مستويات النصوص في التطبيق.

الخصائص الرئيسية:

  • variant: تحدد نمط النص (h1, h2, subtitle1, body1, ...)
  • align: محاذاة النص (right, left, center, justify)
  • color: لون النص (primary, secondary, textPrimary, ...)
  • sx: خاصية للتخصيص المتقدم باستخدام نظام MUI
عنوان رئيسي (h1)
عنوان ثانوي (h2)
نص فرعي (subtitle1)
هذا نص عادي للجسم (body1)
نص صغير للجسم (body2)

أمثلة على أنواع النصوص في MUI

كود الاستخدام:

import Typography from '@mui/material/Typography';

function App() {
  return (
    <div>
      <Typography variant="h1">عنوان رئيسي</Typography>
      <Typography variant="h2" color="primary">عنوان ثانوي</Typography>
      <Typography variant="subtitle1" align="center">نص فرعي</Typography>
      <Typography variant="body1" sx={{ fontWeight: 'bold' }}>
        نص عادي مع تخصيص
      </Typography>
    </div>
  );
}

التنسيق في MUI

توفر MUI عدة طرق للتنسيق والتخصيص، بدءًا من CSS التقليدي وحتى نظام sx المتقدم.

طرق التنسيق:

  • CSS التقليدي: استخدام ملفات CSS/SCSS
  • خاصية sx: تخصيص مباشر باستخدام كائن JavaScript
  • styled API: إنشاء مكونات مخصصة باستخدام styled
  • Theme: تخصيص شامل باستخدام نظام الثيمات
CSS التقليدي
.my-class { color: blue; }
خاصية sx
sx={{ color: 'purple', fontSize: 20 }}
styled API
const MyButton = styled(Button)({ ... });

مقارنة طرق التنسيق:

الطريقة التعقيد الأداء المرونة أفضل استخدام
CSS التقليدي منخفض ممتاز منخفضة تطبيقات بسيطة
خاصية sx متوسط جيد عالية تخصيصات سريعة
styled API مرتفع جيد جداً عالية جداً مكونات مخصصة متكررة

System Properties

نظام الخصائص في MUI يوفر مجموعة من الأدوات للتنسيق السريع والفعال باستخدام خاصية sx. يمكنك التحكم في الهوامش، الحشو، الألوان، والمزيد بسهولة.

أهم الخصائص:

التخطيط

  • m, mt, mr, mb, ml, mx, my (margins)
  • p, pt, pr, pb, pl, px, py (paddings)
  • width, height, display

التصميم

  • color, bgcolor (background-color)
  • border, borderRadius
  • boxShadow

الطباعة

  • fontFamily, fontSize
  • fontWeight, fontStyle
  • textAlign, letterSpacing

مثال على خاصية sx:

<Box
  sx={{
    p: 2,                    // padding: 16px
    m: 1,                    // margin: 8px
    bgcolor: 'primary.main', // background-color
    color: 'white',
    borderRadius: 2,         // 8px
    boxShadow: 3,            // theme.shadows[3]
    '&:hover': {
      bgcolor: 'primary.dark',
      boxShadow: 6,
    }
  }}
>
  صندوق مع تخصيص متقدم
</Box>

Buttons - الأزرار

توفر MUI عدة أنواع من الأزرار لتلبية مختلف احتياجات التطبيق. الأزرار هي من أكثر المكونات استخدامًا في أي واجهة مستخدم.

أنواع الأزرار:

Button

IconButton

TextField

كود Button:

import Button from '@mui/material/Button';
import IconButton from '@mui/material/IconButton';
import TextField from '@mui/material/TextField';
import SearchIcon from '@mui/icons-material/Search';

function App() {
  return (
    <div>
      <Button variant="contained" color="primary">
        أساسي
      </Button>
      
      <IconButton color="secondary">
        <FavoriteIcon />
      </IconButton>
      
      <TextField 
        placeholder="ابحث..."
        InputProps={{
          startAdornment: (
            <InputAdornment position="start">
              <SearchIcon />
            </InputAdornment>
          )
        }}
      />
    </div>
  );
}

الخصائص الشائعة:

  • variant: contained | outlined | text
  • color: primary | secondary | success | error
  • size: small | medium | large
  • disabled: تعطيل الزر
  • startIcon/endIcon: إضافة أيقونات

مكون Box - الحاوية المتعددة الاستخدامات

مكون Box في MUI يعتبر حاوية متعددة الاستخدامات تعمل كعنصر غلاف لتطبيق التصميمات. يعمل كبديل لعناصر div مع إضافة خاصية sx للتخصيص السهل.

الاستخدامات الرئيسية:

  • حاوية لتطبيق الأنماط باستخدام خاصية sx
  • بديل لـ div مع دعم نظام خاصية sx
  • تجميع العناصر وتطبيق التصميمات المشتركة
  • إنشاء تخطيطات بسيطة بدون استخدام CSS

مزايا استخدام Box:

  • سهولة التخصيص مباشرة في الكود
  • دعم كامل لنظام التصميم في MUI
  • مرونة في تطبيق التصميمات
  • بديل بسيط للعناصر div العادية

مكون Container - الحاوية المنظمة

مكون Container يوفر حاوية مركزة تتحكم في عرض المحتوى بشكل متجاوب مع أحجام الشاشات المختلفة. يعمل على تنظيم المحتوى بشكل احترافي.

خصائص أساسية:

  • maxWidth: يحدد أقصى عرض للحاوية (xs, sm, md, lg, xl)
  • fixed: يثبت عرض الحاوية بدلاً من التجاوب
  • disableGutters: يزيل الهوامش الجانبية
  • يدعم خاصية sx للتخصيص الإضافي

متى تستخدم Container؟

  • عندما تحتاج إلى محتوى مركزي في الصفحة
  • لإنشاء تخطيط متجاوب يتحجم مع الشاشة
  • عندما تريد هوامش جانبية متسقة
  • لتنظيم المحتوى الرئيسي للتطبيق

أمثلة عملية:

حاوية بعرض كامل (fluid):
محتوى الحاوية
حاوية متوسطة (md):
محتوى الحاوية
حاوية صغيرة (sm):
محتوى الحاوية

كود الاستخدام:

import Container from '@mui/material/Container';

function App() {
  return (
    // حاوية متوسطة مع هوامش
    <Container maxWidth="md">
      <Box sx={{ bgcolor: 'primary.main', color: 'white', p: 2 }}>
        محتوى في حاوية متوسطة
      </Box>
    </Container>
    
    // حاوية بعرض كامل بدون هوامش
    <Container maxWidth={false} disableGutters>
      <Box sx={{ bgcolor: 'secondary.main', color: 'white', p: 2 }}>
        محتوى في حاوية كاملة العرض
      </Box>
    </Container>
    
    // حاوية ثابتة (غير متجاوبة)
    <Container fixed>
      <Box sx={{ bgcolor: 'warning.main', p: 2 }}>
        محتوى في حاوية ثابتة
      </Box>
    </Container>
  );
}

مكون Stack - الترتيب الذكي

مكون Stack يوفر طريقة سهلة لترتيب العناصر في اتجاه واحد (عمودي أو أفقي) مع التحكم في المسافات بينها. هو بديل ممتاز لاستخدام CSS Flexbox مباشرة.

الخصائص الرئيسية:

  • direction: اتجاه الترتيب (row, column, row-reverse, column-reverse)
  • spacing: المسافة بين العناصر (بالنقاط 1 = 8px)
  • divider: إضافة فاصل بين العناصر
  • justifyContent: محاذاة العناصر على المحور الرئيسي
  • alignItems: محاذاة العناصر على المحور الثانوي

مزايا استخدام Stack:

  • بديل سهل لـ Flexbox
  • تحكم دقيق في المسافات بين العناصر
  • دعم للاتجاهات المختلفة بسهولة
  • إمكانية إضافة فواصل بين العناصر
  • تكامل سلس مع مكونات MUI الأخرى

أمثلة عملية:

Stack عمودي:
عنصر 1
عنصر 2
عنصر 3
Stack أفقي:
عنصر 1
عنصر 2
عنصر 3
Stack مع فواصل:
عنصر 1
عنصر 2
عنصر 3

كود الاستخدام:

import Stack from '@mui/material/Stack';
import Divider from '@mui/material/Divider';

function App() {
  return (
    // Stack عمودي مع مسافات
    <Stack direction="column" spacing={2}>
      <Box sx={{ bgcolor: 'primary.main', p: 2 }}>عنصر 1</Box>
      <Box sx={{ bgcolor: 'primary.main', p: 2 }}>عنصر 2</Box>
      <Box sx={{ bgcolor: 'primary.main', p: 2 }}>عنصر 3</Box>
    </Stack>
    
    // Stack أفقي مع محاذاة مركزية
    <Stack direction="row" spacing={3} justifyContent="center" alignItems="center">
      <Box sx={{ bgcolor: 'secondary.main', p: 2 }}>عنصر أ</Box>
      <Box sx={{ bgcolor: 'secondary.main', p: 2 }}>عنصر ب</Box>
      <Box sx={{ bgcolor: 'secondary.main', p: 2 }}>عنصر ج</Box>
    </Stack>
    
    // Stack عمودي مع فواصل
    <Stack direction="column" divider={<Divider orientation="horizontal" flexItem />}>
      <Box sx={{ p: 2 }}>قسم 1</Box>
      <Box sx={{ p: 2 }}>قسم 2</Box>
      <Box sx={{ p: 2 }}>قسم 3</Box>
    </Stack>
  );
}

مقارنة بين مكونات التخطيط

كل مكون من مكونات التخطيط في MUI له استخداماته المحددة. الفهم الجيد للفروقات بينهم يساعد في اختيار المكون المناسب لكل موقف.

المكون الغرض الأساسي مزايا رئيسية متى تستخدم متى تتجنب
Box حاوية متعددة الاستخدامات لتطبيق التصميمات مرنة، تدعم sx، بديل لـ div لتطبيق أنماط على مجموعة عناصر عند الحاجة لتخطيط معقد
Container تنظيم عرض المحتوى وتحديد أقصى عرض تجاوبي، يتحكم في الهوامش، أحجام محددة للمحتوى الرئيسي في الصفحة للمكونات الصغيرة داخل الصفحة
Stack ترتيب العناصر في اتجاه واحد سهولة التحكم في المسافات، اتجاهات متعددة لترتيب مجموعة عناصر في صف أو عمود عند الحاجة لتخطيط ثنائي الأبعاد

كيفية اختيار المكون المناسب

استخدم Box عندما:

  • تحتاج إلى تطبيق أنماط CSS على مجموعة عناصر
  • تريد بديلاً لـ div مع دعم خاصية sx
  • تحتاج إلى حاوية بسيطة بدون سلوك معقد

استخدم Container عندما:

  • تريد تنظيم المحتوى الرئيسي في الصفحة
  • تحتاج إلى تحديد أقصى عرض للمحتوى
  • تريد هوامش جانبية متسقة على مختلف الشاشات

استخدم Stack عندما:

  • تحتاج إلى ترتيب عناصر في صف أو عمود
  • تريد التحكم في المسافات بين العناصر
  • تحتاج إلى إضافة فواصل بين العناصر
  • تريد بديلاً سهلًا لـ Flexbox

💡 النصيحة الاحترافية: يمكنك دمج هذه المكونات معًا! استخدم Container للهيكل العام، Stack لترتيب العناصر، و Box لتطبيق التصميمات التفصيلية.
