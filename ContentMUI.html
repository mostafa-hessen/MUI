Introduction to Material UI

Material UI (MUI) is a popular React UI framework that implements Google's Material Design principles.

Why Material UI?

  • Beautiful, consistent UI components out of the box
  • Comprehensive component library (80+ components)
  • Customizable theming system
  • Excellent documentation and community support
  • Accessibility built-in
  • Optimized for React development

Strengths

  • Rich component library
  • Strong theming capabilities
  • Professional design aesthetics
  • Regular updates and maintenance

Considerations

  • Larger bundle size
  • Learning curve for customization
  • Material Design might not fit all projects

UI Library Comparison

Material UI

  • Complete Material Design implementation
  • Powerful theming system
  • Extensive component library
  • Larger bundle size

Bootstrap

  • Widely used with extensive resources
  • Responsive grid system
  • Generic look without customization
  • Not designed specifically for React

Tailwind CSS

  • Highly customizable utility-first approach
  • Smaller bundle size with purging
  • No pre-built components
  • Steeper learning curve

When to Choose Material UI

Enterprise Applications

When you need a professional, consistent UI across a large application

Rapid Prototyping

When you need to build functional prototypes quickly

Material Design Projects

When your project requires Google's Material Design spec

Team Collaboration

When working with a team that benefits from standardized components

Setup and Configuration

Installation

npm install @mui/material @emotion/react @emotion/styled

Font Setup

Add to your HTML head:

<link rel="stylesheet" href="https://fonts.googleapis.com/css?family=Roboto:300,400,500,700&display=swap"/>
<link rel="stylesheet" href="https://fonts.googleapis.com/icon?family=Material+Icons"/>

Basic Setup

// App.js
import React from 'react';
import { Button } from '@mui/material';
import { ThemeProvider, createTheme } from '@mui/material/styles';

const theme = createTheme();

function App() {
  return (
    <ThemeProvider theme={theme}>
      <Button variant="contained">Hello World</Button>
    </ThemeProvider>
  );
}

export default App;

Lab: Build a Navbar

Objective

Build a responsive navigation bar with:

  • App bar with logo and title
  • Navigation links
  • Mobile menu with hamburger icon
  • Responsive behavior

Steps

  1. Install Material UI and dependencies
  2. Set up theme and fonts
  3. Create AppBar component
  4. Add Toolbar with logo and title
  5. Create navigation links (use Box for spacing)
  6. Implement mobile menu with Drawer
  7. Add responsive behavior with useMediaQuery
  8. Style with theme colors

Code Solution

import React, { useState } from 'react';
import { 
  AppBar, Toolbar, Typography, Box, 
  IconButton, Drawer, List, ListItem 
} from '@mui/material';
import MenuIcon from '@mui/icons-material/Menu';

function Navbar() {
  const [mobileOpen, setMobileOpen] = useState(false);
  
  const handleDrawerToggle = () => {
    setMobileOpen(!mobileOpen);
  };
  
  const navItems = ['Home', 'About', 'Services', 'Contact'];
  
  return (
    <>
      <AppBar position="static">
        <Toolbar>
          <Typography variant="h6" sx={{ flexGrow: 1 }}>
            MyApp
          </Typography>
          <Box sx={{ display: { xs: 'none', sm: 'block' } }}>
            {navItems.map((item) => (
              <Button key={item} color="inherit">{item}</Button>
            ))}
          </Box>
          <IconButton
            color="inherit"
            onClick={handleDrawerToggle}
            sx={{ display: { sm: 'none' } }}
          >
            <MenuIcon />
          </IconButton>
        </Toolbar>
      </AppBar>
      
      <Drawer
        variant="temporary"
        open={mobileOpen}
        onClose={handleDrawerToggle}
      >
        <List>
          {navItems.map((item) => (
            <ListItem key={item}>{item}</ListItem>
          ))}
        </List>
      </Drawer>
    </>
  );
}

export default Navbar;

Best Practices

  • Use ThemeProvider for consistent styling
  • Break components into smaller reusable parts
  • Use sx prop for responsive styles
  • Implement accessibility features
  • Separate concerns (create separate Navbar component)
